# AiNiee Product UI Blueprint v3.0

## 1. 产品概述

AiNiee 是一款AI驱动的翻译工具，旨在为开发者和翻译团队提供智能化的翻译解决方案。通过集成先进的AI模型和直观的用户界面，帮助用户高效完成多语言内容翻译和本地化工作。

产品核心价值：从开发者工具向专业翻译平台演进，提供Smartcat级别的专业翻译管理体验。

## 2. 竞品深度对比：Smartcat vs AiNiee

### 2.1 Smartcat 核心优势分析

| 功能模块 | Smartcat | AiNiee 现状 | 差距分析 |
|---------|----------|------------|----------|
| **CAT网格编辑器** | ✓ 专业级可视化网格，支持原文/译文对照编辑 | ✗ 无可视化编辑器，仅文本输出 | 核心功能缺失 |
| **项目化管理** | ✓ 统一项目工作流，支持多文件、多语言管理 | ✗ 文件路径依赖，无项目概念 | 架构设计差异 |
| **AI+人工工作流** | ✓ AI翻译+人工审校+自动化流程 | ✗ 纯AI翻译，无人工协作机制 | 工作流完整性 |
| **进度可视化** | ✓ 文件级进度条、状态标识、实时更新 | ✗ 日志驱动，无进度可视化 | 用户体验差距 |
| **团队协作** | ✓ 角色权限管理、任务分配、评论系统 | ✗ 单人工具，无协作功能 | 团队功能缺失 |
| **翻译记忆库** | ✓ 内置TM、术语库、自动匹配 | ✗ 无翻译记忆功能 | 专业性不足 |

### 2.2 Smartcat UI设计亮点

1. **现代化界面**：简洁的白色主题，卡片式布局，视觉层次分明
2. **可视化进度**：文件列表配合进度条，状态一目了然
3. **网格编辑器**：专业的CAT编辑器，支持段落级对照编辑
4. **工作流可视化**：AI+人工流程的图形化配置界面
5. **响应式设计**：适配多设备，触控友好的交互设计

## 3. AiNiee v3.0 演进路线

### 3.1 阶段一：UI现代化（当前阶段）

**已完成**：
- Glassmorphism设计风格
- 控制台/仪表盘布局优化
- 基础组件现代化

**待完善**：
- 色彩系统标准化
- 交互动画优化
- 响应式布局增强

### 3.2 阶段二：项目化重构（近期目标）

**核心目标**：引入"项目(Project)"概念，替代文件路径依赖

**功能规划**：
1. **项目创建与管理**
   - 项目名称、描述、目标语言设置
   - 项目模板和预设配置
   - 项目级统计和报告

2. **文件列表视图**
   - 可视化文件管理界面
   - 文件状态追踪（待翻译/翻译中/已完成）
   - 批量操作和筛选功能

3. **进度可视化组件**
   ```
   | 文件名 | 语言 | 进度条 | 状态 | 操作 |
   |--------|------|--------|------|------|
   | main.json | EN→CN | ████████░░ 80% | 翻译中 | [预览][下载] |
   | config.ts | EN→JP | ██████████ 100% | 已完成 | [预览][下载] |
   ```

### 3.3 阶段三：Visual CAT Editor（中期目标）

**核心目标**：构建可视化网格编辑器，达到Smartcat专业水准

**功能规格**：
1. **网格编辑器核心**
   - 原文/译文对照显示
   - 段落级编辑和导航
   - 实时保存和版本控制

2. **AI辅助功能**
   - 段落级AI重译
   - 翻译建议和质量评分
   - 术语一致性检查

3. **协作功能**
   - 评论和批注系统
   - 版本对比和合并
   - 角色权限管理

## 4. 详细UI设计规范

### 4.1 设计系统

**色彩方案**：
- 主色：#3B82F6 (蓝色) - 主要操作和品牌色
- 辅助色：#10B981 (绿色) - 成功状态
- 警告色：#F59E0B (橙色) - 警告和进行中
- 错误色：#EF4444 (红色) - 错误和失败
- 中性色：#6B7280 (灰色) - 文本和边框

**字体系统**：
- 标题：Inter Bold, 24-32px
- 正文：Inter Regular, 14-16px
- 小字：Inter Regular, 12px

**组件风格**：
- 按钮：圆角8px，悬停阴影效果
- 卡片：圆角12px，轻微阴影
- 输入框：圆角6px，聚焦边框高亮

## 5. 技术实现优先级

### 5.1 第一阶段（1-2周）
- 项目创建和管理界面
- 文件列表视图组件

### 5.2 第二阶段（3-4周）
- 基础CAT编辑器框架
